# MultiWaveForm サンプル

マイクやオーディオインターフェースなどから入力した音声を、**リアルタイムで波形**として表示するサンプルです。  
初心者の方が「音声入力 → 波形表示」の流れを理解するための見本として使えます。

---

## このサンプルでできること

- PC に接続した**入力デバイス（マイクなど）**から音声を取得する
- 取得した音声を**波形グラフ**として画面に描画する
- **複数チャンネル**（例：ステレオなら L/R 2本）を、色分けして同時に表示できる
- チャンネルごとに**表示の ON/OFF** を切り替えられる

---

## 必要な環境

- **Unity**（本プロジェクトの推奨バージョンに合わせてください）
- **Lasp** プラグイン（本プロジェクトの `02. Audio / Lasp` などで提供されている想定です）
- **オーディオ入力デバイス**（マイク、オーディオインターフェースなど）が PC に接続され、Unity から使える状態になっていること

---

## 使い方（最短手順）

1. **シーンを開く**  
   `[Scene] MultiWaveForm.unity` をダブルクリックして開く。

2. **再生する**  
   エディタ上部の **Play** ボタンを押す。

3. **音を入れる**  
   マイクに話しかけたり、オーディオインターフェースから音を送ったりすると、画面上に波形が表示されます。

4. **コンソールでデバイスを確認**  
   再生開始時に、利用可能な入力デバイスが  
   `[MultiWaveData] 0: デバイス名`  
   のようにコンソールに表示されます。  
   「採用: インデックス ○, デバイス「○○」, チャンネル数 ○」で、実際に使っているデバイスとチャンネル数が分かります。

---

## シーンの構成（何がどこにあるか）

このサンプルは、主に **2 つのスクリプト** で動いています。

| 役割 | スクリプト | 何をしているか（イメージ） |
|------|------------|----------------------------|
| **音声を集める** | **MultiWaveData** | 入力デバイスを選び、マイクなどの音声を「波形の数値の並び」として内部に蓄えます。 |
| **波形を描く** | **WaveDrawer** | MultiWaveData が蓄えた数値をもらって、Unity の LineRenderer で画面上に線（波形）として描画します。 |

- シーン内の **同じ GameObject** に、**MultiWaveData** と **WaveDrawer** の両方がアタッチされているか、  
  または **WaveDrawer** の「Data Source」に **MultiWaveData** が入っているオブジェクトを指定する形になっていることが多いです。
- **WaveDrawer** は「Line Material」に、サンプル同梱の **Material（例: LineMaterial_Addictive）** を指定する必要があります。

---

## データの流れ（3ステップ）

1. **MultiWaveData**  
   入力デバイスから音声を取得し、一定時間分の波形データ（float の配列）を保持する。

2. **WaveDrawer**  
   MultiWaveData から「○秒分・○点」の波形データ（`GetWaveData`）をもらう。

3. **WaveDrawer**  
   その数値を「横軸＝時間・縦軸＝振幅」の座標に変換し、**LineRenderer.SetPositions** で線として描画する。

「**データを取得 → 座標に変換 → 描画**」という流れだけ押さえておくと、コードを読むときも分かりやすくなります。

## インスペクタの主な設定（MultiWaveData）

| 項目 | 説明（初心者向け） |
|------|--------------------|
| **デバイス** | 使う入力デバイスの**番号**。0 が一覧の先頭（多くはデフォルトのマイク）です。コンソールの「0: デバイス名」の番号と対応します。 |
| **バッファ（最大で保持する秒数）** | 何秒分までの音声を蓄えておくか。WaveDrawer で表示する「何秒分」は、この範囲内で設定します。 |

---

## インスペクタの主な設定（WaveDrawer）

| 項目 | 説明（初心者向け） |
|------|--------------------|
| **Data Source** | **必須。** 波形データの元になる **MultiWaveData** を指定します。ここが空だとエラーになり、波形は描画されません。 |
| **Sample Seconds** | 波形として表示する「何秒分」のデータを取得するか。短いほど直近の音の変化が分かりやすくなります。 |
| **Draw Resolution** | その何秒分を「何個の点」で表すか。数が大きいほど滑らかな線になりますが、重くなりやすいです。 |
| **Display Width / Height** | 波形が描かれる範囲の**幅・高さ**（ワールド単位）。カメラやキャンバスのスケールに合わせて調整します。 |
| **Line Width** | 波形の線の太さ。 |
| **Amplitude** | 波形の**縦方向の大きさ**（振幅の倍率）。大きくすると波がはっきり見えます。 |
| **Line Material** | **必須。** 線を描くときに使う **Material**。未指定だとエラーになります。サンプルでは `Material/LineMaterial_Addictive` などを指定します。 |
| **Draw Channel** | **どのチャンネルを表示するか**の bool リスト。インデックスがチャンネル番号に対応し、チェックを付けたチャンネルだけ波形が描画されます。 |
| **色（チャンネルごと）** | チャンネルごとの線の色。1 番目が左チャンネル、2 番目が右チャンネルなどに対応します。 |

## よくあるトラブルと確認ポイント

### 波形が表示されない

1. **Data Source** に MultiWaveData が指定されているか確認する。
2. **Line Material** にマテリアルが指定されているか確認する。
3. **コンソール** に  
   `[MultiWaveData] 採用: インデックス ○, デバイス「○○」, チャンネル数 ○`  
   が出ているか確認する。出ていなければ、入力デバイスが選ばれていません。
4. **デバイス番号**  
   Windows のサウンド設定や Unity のオーディオ設定で、使いたいマイクが「規定のデバイス」になっているか、または MultiWaveData の「デバイス」の番号を、コンソールに表示された一覧から選び直してみる。
5. **Draw Channel**  
   WaveDrawer の「Draw Channel」で、表示したいチャンネルにチェックが入っているか確認する。

### コンソールにエラーが出る

- **「Data Source に MultiWaveData を指定してください」**  
  → WaveDrawer の **Data Source** に、MultiWaveData がアタッチされているオブジェクトを指定する。
- **「Line Material を指定してください」**  
  → WaveDrawer の **Line Material** に、`Material/LineMaterial_Addictive` などのマテリアルを指定する。
- **「MultiWaveData にチャンネルがありません」**  
  → 入力デバイスが認識されていないか、デバイス番号がずれている可能性があります。コンソールのデバイス一覧と「採用」のログを確認し、デバイス番号を変更してみる。

### 再生開始時にデバイス一覧が空 / 採用されない

- マイクの**プライバシー設定**（OS でアプリにマイク許可が出ているか）を確認する。
- **オーディオドライバ**が正しく入っているか、別のアプリでマイクが使えているか確認する。
- Unity を**再起動**してから、もう一度 Play してみる。

## まとめ

- **MultiWaveData** ＝ 入力デバイスから音声を取って「波形の数値」として蓄える。
- **WaveDrawer** ＝ その数値をもらって「線」として描く。  
- 必須設定は **Data Source（MultiWaveData）** と **Line Material**。  
- 動かないときはコンソールの **エラーログ** と **デバイス採用ログ** をまず確認すると原因を絞りやすいです。

疑問や不具合があれば、この README とコンソールのメッセージを手がかりに、設定とシーン構成を確認してみてください。
