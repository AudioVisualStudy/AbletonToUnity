# WaveTexture デモについて

マイクやオーディオインターフェースから入力した音声を、**白黒の縦ストライプ**としてリアルタイムで表示するサンプルです。  
MultiWaveForm が「折れ線」で波形を描くのに対し、WaveTexture は**振幅の正負に応じてストライプの色（白/黒）**を変えることで、音の強さと向きをパターンとして見せます。初心者の方が「波形データ → メッシュ描画」の流れを理解するための見本として使えます。

---

## このサンプルでできること

- PC に接続した**入力デバイス（マイクなど）**から音声を取得する（**MultiWaveData** を使用）
- 取得した波形を**縦方向のストライプ**として描画する（振幅が正のときと負のときで白/黒を切り替え）
- **複数チャンネル**（最大8）を、**表示するチャンネルだけ**を選んで、画面を縦に分割した**帯**として並べて表示できる
- チャンネルごとに**表示 ON/OFF** と**レベルしきい値**を個別に設定できる
- オプションで**上下2分割**（エリアA・エリアB、B は白黒反転）にできる
- デバッグ用に**波形の折れ線オーバーレイ**を各チャンネルの帯の中心に表示できる

---

## 必要な環境

- **Unity**（本プロジェクトの推奨バージョンに合わせてください）
- **Lasp** プラグイン（本プロジェクトの `02. Audio / Lasp` で提供）
- **MultiWaveData** が使う**オーディオ入力デバイス**（マイク、オーディオインターフェースなど）が PC に接続され、Unity から使える状態になっていること

---

## 使い方（最短手順）

1. **シーンを開く**  
   `[Scene] WaveTexture.unity` をダブルクリックして開く。

2. **再生する**  
   エディタ上部の **Play** ボタンを押す。

3. **音を入れる**  
   マイクに話しかけたり、オーディオインターフェースから音を送ったりすると、画面上に**白黒のストライプ**が表示されます。音が大きいほどはっきり白/黒になり、小さいとデフォルト色（グレーなど）になります。

4. **コンソールでデバイスを確認**  
   再生開始時に、MultiWaveData が採用した入力デバイスが  
   `[MultiWaveData] 採用: インデックス ○, デバイス「○○」, チャンネル数 ○`  
   のようにコンソールに表示されます。

---

## シーンの構成（何がどこにあるか）

このサンプルは、**MultiWaveData**（音声を蓄える）と **WaveTexture**（ストライプを描く）の **2 つのコンポーネント**で動いています。

| 役割 | コンポーネント | 何をしているか（イメージ） |
|------|----------------|----------------------------|
| **音声を集める** | **MultiWaveData** | 入力デバイスを選び、音声を「波形の数値の並び」として内部に蓄えます。 |
| **ストライプを描く** | **WaveTexture** | MultiWaveData から波形データ（`GetWaveData`）をもらい、振幅に応じて**白/黒の縦ストライプ**を **Mesh** で描画します。 |

- シーン内の **同じ GameObject** に、**MultiWaveData** と **WaveTexture** の両方がアタッチされているか、または **WaveTexture** の「Data Source」に **MultiWaveData** が入っているオブジェクトを指定する形になっています。
- WaveTexture は実行時に**子オブジェクト**を自動で作ります。
  - **StripesMesh** … ストライプ用の Mesh（MeshFilter + MeshRenderer）
  - **WaveformOverlay** … デバッグ用の波形折れ線（Show Waveform Overlay を ON にしたときだけ表示）
- **Stripe Material** に、サンプル同梱の **StripeMaterial**（頂点色を表示する Unlit マテリアル）を指定する必要があります。

---

## データの流れ（3ステップ）

1. **MultiWaveData**  
   入力デバイスから音声を取得し、一定時間分の波形データ（float の配列）を保持する。

2. **WaveTexture**  
   MultiWaveData から「○秒分・○点」の波形データ（`GetWaveData(チャンネル, 秒数, 解像度)`）をもらう。

3. **WaveTexture**  
   各サンプル点の**振幅**を見て、  
   - そのフレームの**チャンネル全体のレベル（ピーク振幅）**が**レベルしきい値以上**なら … 振幅が正 → 白、負 → 黒（オプションで上下分割時は B 側は反転）  
   - しきい値未満なら … **デフォルト色**（グレーなど）  
   にし、その色の**四角形（ストライプ1本）**の頂点と色を Mesh に書き込み、描画する。

「**データを取得 → レベルと正負で色を決める → メッシュで描画**」という流れです。

## インスペクタの主な設定（MultiWaveData）

| 項目 | 説明（初心者向け） |
|------|--------------------|
| **デバイス** | 使う入力デバイスの**番号**。0 が一覧の先頭（多くはデフォルトのマイク）です。 |
| **バッファ（最大で保持する秒数）** | 何秒分までの音声を蓄えておくか。WaveTexture の「Sample Seconds」はこの範囲内で設定します。 |

---

## インスペクタの主な設定（WaveTexture）

| 項目 | 説明（初心者向け） |
|------|--------------------|
| **Data Source** | **必須。** 波形データの元になる **MultiWaveData** を指定します。空だとエラーになり、ストライプは描画されません。 |
| **Sample Seconds** | 波形として使う「何秒分」のデータを取得するか。短いほど直近の音の変化がストライプに反映されやすくなります。 |
| **Draw Resolution** | その何秒分を「何本のストライプ」で表すか。数が大きいほど細かくなりますが、重くなりやすいです。 |
| **Draw Rect** | ストライプが描かれる**矩形の範囲**（ローカル座標）。X,Y が左・下の端、Width/Height が幅・高さです。 |
| **Stripe Material** | **必須。** ストライプを描くときに使う **Material**。頂点色を表示する Unlit マテリアル（サンプルの StripeMaterial）を指定します。 |
| **Default Color** | レベルがしきい値**未満**のときに使う色（例：グレー）。音が小さいときのストライプの色です。 |
| **チャンネル** | 最大8個。各要素で **表示**（このチャンネルを描画するか）と **Level Threshold**（このチャンネルのレベルしきい値）を指定します。表示にチェックを入れたチャンネルだけが、上から縦に帯状に並びます。 |
| **Split Area A/B** | ON にすると、各チャンネルの帯を**上下2分割**します。上＝エリアA（正→白・負→黒）、下＝エリアB（正→黒・負→白）です。 |
| **Show Waveform Overlay** | デバッグ用。ON にすると、各チャンネルの帯の**中心**に波形の折れ線が表示されます。**Waveform Line Material** などの指定が必要です。 |

## よくあるトラブルと確認ポイント

### ストライプが表示されない

1. **Data Source** に MultiWaveData が指定されているか確認する。
2. **Stripe Material** にマテリアルが指定されているか確認する。
3. **チャンネル**の **表示** に、少なくとも 1 チャンネル以上チェックが入っているか確認する。
4. **コンソール** に  
   `[MultiWaveData] 採用: インデックス ○, デバイス「○○」, チャンネル数 ○`  
   が出ているか確認する。出ていなければ、入力デバイスが選ばれていません。
5. **Draw Rect** の範囲がカメラから見える位置・大きさになっているか確認する。

### コンソールにエラーが出る

- **「Data Source に MultiWaveData を指定してください」**  
  → WaveTexture の **Data Source** に、MultiWaveData がアタッチされているオブジェクトを指定する。
- **「Stripe Material を指定してください」**  
  → WaveTexture の **Stripe Material** に、`Material+Shader/StripeMaterial` などを指定する。
- **「MultiWaveData にチャンネルがありません」**  
  → 入力デバイスが認識されていないか、デバイス番号がずれている可能性があります。MultiWaveData のデバイス番号を、コンソールのデバイス一覧から選び直してみる。

### 全部同じ色（白黒が出ない）になる

- 各チャンネルの **Level Threshold** が高すぎると、ほとんどが「しきい値未満」になり Default Color ばかりになります。値を**下げる**（例：0.01 や 0.001）と、小さい音でも白/黒が出やすくなります。

```
[Sample] WaveTexture/
├── WaveTextureデモについて.md   ← このファイル

├── Script/
```

※ 波形データの取得は **MultiWaveData** が行うため、MultiWaveForm サンプルと同じく、シーンに MultiWaveData が含まれているか、別シーンから参照できるようにしてください。MultiWaveData の詳しい使い方は **`[Sample] MultiWaveForm/MultiWaveDataの使い方.md`** を参照してください。

## まとめ

- **MultiWaveData** ＝ 入力デバイスから音声を取って「波形の数値」として蓄える。
- **WaveTexture** ＝ その数値をもらい、振幅の正負とレベルしきい値に応じて**白黒の縦ストライプ**を Mesh で描く。
- 必須設定は **Data Source（MultiWaveData）** と **Stripe Material**。表示したいチャンネルは **チャンネル** の **表示** にチェックを入れる。
- 動かないときはコンソールの **エラーログ** と **デバイス採用ログ** をまず確認すると原因を絞りやすいです。

疑問や不具合があれば、このドキュメントとコンソールのメッセージを手がかりに、設定とシーン構成を確認してみてください。
